A banner

Same line the current build draws (▸ /v ● Recording) — extended with a tabular timer, then live device · language · model · vad chips. Chips render as `[k v]` cells in plain ANSI, so the layout still fits a 24-row terminal.

B meter rack

Left: spectrogram drawn with `▁▂▃▄▅▆▇█` cells in 8 rows, color-mapped per band — purely a render of the existing FFT we already compute for VU.
Right: the original 20-bar VU kept verbatim, plus a 4-cell readout (RMS · PEAK · SR · SEG) sourced from values already in `voice.ts`.

C waveform & transcript

A single-row 5 s scrolling waveform built from the same Int16 buffer the engine consumes — one extra reduce, no extra capture. Transcript marks low-confidence words with a soft amber underline and tags code-y tokens in cyan.

D footer

Adds discoverable bindings without losing the originals: ⌥↵ paste+send, ⌥L language, ⌥E edit-before-paste. Right side is a quiet perf stamp — cheap honesty so the user trusts what is happening.

Cyan carries signal & success; crimson is reserved exclusively for recording; mauve marks language/auto-detect; yellow appears only when paused. Backgrounds stay near-black so the UI sits under Pi's chat without competing.

budget

All new visuals reuse data the pipeline already produces. We're not asking for new capture — only smarter render of the same Int16 LE buffer.

spectrogram 1× FFT-512 per 50 ms = 20 Hz; 8 band reduce; ring buffer 60 columns ~0.3% CPU
waveform ribbon min/max reduce of the same Int16 chunk decibri already delivers free
word-conf colors sherpa-onnx exposes per-token probabilities; map to low/mid/ok existing field
history strip append-only; 3-row LRU; same overlay scrolls instead of redrawing ~3 KB ram
terminal floor full layout fits 80×24; if rows<20 we drop spectrogram + history first graceful
