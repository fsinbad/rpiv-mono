# Research Questions: rpiv-pi Current-State Audit

## Discovery Summary

Five parallel discovery agents surveyed rpiv-pi's extension code (6 TS files in `extensions/rpiv-core/`), 17 skills in `skills/`, 9 agent profiles in `agents/`, 5 guidance files in `.rpiv/guidance/`, sibling-plugin integration surface, and 72 historical artifacts in `thoughts/shared/`. Five key files were read for structural depth: `extensions/rpiv-core/index.ts` (entry + 5 hooks + 2 commands), `agents.ts` (manifest-based sync engine), `guidance.ts` (per-depth AGENTS/CLAUDE/architecture resolver), `git-context.ts` (cached branch/commit/user injection), and `skills/discover/SKILL.md` as a representative skill. The plugin has successfully extracted four tool surfaces (`ask_user_question`, `todo`, `advisor`, `web_search`/`web_fetch`) plus subagent primitives to sibling Pi plugins, leaving rpiv-core as a pure-infrastructure extension owning guidance injection, git context, thoughts/ scaffolding, bundled-agent sync, and a `/rpiv-setup` bootstrapper. Post-extraction drift is visible in two guidance files, one `.pi/agents/CLAUDE.md`, thin tool-name discipline in skills (only `code-review` names the `todo` tool explicitly), and one sibling (`rpiv-advisor`) that is marked `required` in `/rpiv-setup` but called by zero skills.

Historical artifacts group into five threads that bear directly on this audit: sibling-plugin extraction (primary thread, plans dated 2026-04-13), guidance-injection refactor (CLAUDE.md subfolder resolution, 2026-04-13), agent-sync refactor (manifest system, 2026-04-14), naming-conventions unification (2026-04-14), and the README professional rewrite (2026-04-14). No `handoffs/` directory exists. Fourteen plans sit in `status: ready` without explicit completion markers — part of the audit is classifying those as done/partial/abandoned relative to current code.
