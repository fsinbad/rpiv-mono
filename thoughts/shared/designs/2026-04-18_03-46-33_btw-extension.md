# Design: `/btw` Side-Question Slash Command (`@juicesharp/rpiv-btw`)

## Summary

`/btw <question>` is a new sibling Pi extension that asks the same primary model a one-off side question using the cloned primary conversation as context. Built as a standalone advisor-style extension (not a wrapper around `@tintinweb/pi-subagents`). Calls `completeSimple` directly with raw `convertToLlm(branch)` plus per-session-keyed `btwHistory` threading. Renders the answer in a borderless bottom-slot overlay via `ctx.ui.custom`. Storage lives on `globalThis[Symbol.for("rpiv-btw")]` keyed by session-file (cache-friendly) and is mirrored to disk via `pi.appendEntry("btw:sidechain", ...)` for cross-process durability.

## Requirements

- `/btw <question>` dispatches synchronously even mid-stream of the main agent (TUI input editor blocks for the handler; main agent's turn continues independently).
- Same primary model answers the side question (`ctx.model`); no model picker, no persistence of model choice.
- Side answer never enters main agent's `agent.state.messages` — ephemeral display only.
- History of `/btw` Q+A pairs threads across calls within the same session and persists across `/new`, `/fork`, `/reload`, `/resume` within the Pi process.
- Audit record (`btw:sidechain`) survives Pi process exit via JSONL transcript replay.
- Bottom-slot overlay: borderless, `↑/↓` scroll, `x` clear current-session history, `Esc` dismiss + abort in-flight call.
- Fork `f` key deferred to post-MVP.
- Best-effort prompt-cache hit rate (no byte-identical guarantee against competitor; pi-ai's `streamAnthropic` constructs cache breakpoints internally).
- One sibling registration entry in `rpiv-pi/extensions/rpiv-core/siblings.ts` + one `peerDependencies` row in `rpiv-pi/package.json` + README rows for `/btw`.

## Current State Analysis

### Key Discoveries

- **Slash-command dispatch path** routes `/` prefix to `_tryExecuteExtensionCommand` immediately at `agent-session.js:672-681`, even mid-stream. Extension commands cannot be steered or follow-upped (`agent-session.js:922-931`). Handler signature: `(args: string, ctx: ExtensionCommandContext) => Promise<void>` per `types.d.ts:693`.
- **Branch read**: `ctx.sessionManager.getBranch()` returns `SessionEntry[]`; filter `type === "message"`, map to `e.message`, then `convertToLlm(agentMessages)` to get LLM `Message[]` (advisor pattern at `advisor.ts:249-264`; converter at `messages.js:75-122`). Custom messages, branch summaries, compaction summaries become user-role; user/assistant/toolResult pass through unchanged.
- **`sendMessage({display: false})` does NOT hide from LLM context** — only TUI rendering. Confirmed at `interactive-mode.js:2225-2233` (TUI gate) vs `messages.js:89-96` (custom-message case unconditionally emits user-role LLM message). Only safe ephemeral sinks: `ctx.ui.custom` (transient) and `pi.appendEntry` (persisted but not LLM-visible per `session-manager.d.ts:65-69`).
- **Module re-import on `/new`/`/fork`/`/resume`**: confirmed via `loader.js:223-234` (jiti `moduleCache: false`). Lifecycle chain at `agent-session-runtime.js:103-208` calls `teardownCurrent → createRuntime → loadExtensions`. Module-level `let foo = []` resets on each. **Storage on `globalThis` survives this**, mirroring pi-subagents' `globalThis[Symbol.for("pi-subagents:manager")]` at `@tintinweb/pi-subagents/src/index.ts:413-422`.
- **`ctx.model: Model<any> | undefined`** at `types.d.ts:192` — primary model, freshly read each access. No persistence machinery needed (unlike advisor at `advisor.ts:96-125`).
- **`completeSimple(model, {systemPrompt, messages, tools?}, {apiKey, headers, signal, reasoning?})`** signature at `pi-ai/dist/stream.d.ts:7`. Four StopReason branches: `aborted`, `error`, empty text, thrown Error — modeled at `advisor.ts:278-335`.
- **`ctx.modelRegistry.getApiKeyAndHeaders(model)`** returns `{ok: true, apiKey?, headers?} | {ok: false, error}` per `model-registry.d.ts:7-14`. Always `await` — fresh-resolves credentials each call.
- **`ctx.signal` is bound to MAIN AGENT** (`runner.js:1758` → `agent.js:304`). DO NOT forward to `/btw`'s `completeSimple` — coupling main-agent abort to side-call abort. Use own `AbortController` and abort it from the overlay's Esc handler.
- **`ctx.ui.custom<T>(factory, {overlay, overlayOptions})`** signature at `types.d.ts:93-103`. Factory signature: `(tui, theme, keybindings, done) => Component & {dispose?}`. `overlayOptions.anchor` includes `"bottom-center"` per `pi-tui/dist/tui.d.ts:59`. Default `nonCapturing: false` — overlay receives focus and `handleInput` callbacks.
- **Bottom-anchor precedent exists** but only in pi-coding-agent QA examples (`overlay-qa-tests.ts:281, :293`) with `width: <number>` and `margin: {bottom: 1}`. No installed sibling has shipped `width: "100%"` + `margin: {bottom: 0}` — `/btw` would be first.
- **`pi.on("message_end", handler)`** at `types.d.ts:722`. Event payload `{type: "message_end", message: AgentMessage}` at `types.d.ts:443-446`. Narrow on `msg.role === "assistant"` then skip `stopReason === "toolUse"` for cache-safe snapshots — competitor's `handleStopHooks` semantics.
- **`pi.appendEntry(customType, data)`** at `types.d.ts:762-763`: persisted in transcript JSONL, "Does NOT participate in LLM context" per `session-manager.d.ts:65-69`. Mirrors pi-subagents' `subagents:record` audit pattern at `@tintinweb/pi-subagents/src/index.ts:376-380`.
- **SIBLINGS registry** at `rpiv-pi/extensions/rpiv-core/siblings.ts:22-48` — single source of truth. Adding an entry auto-wires presence detection (`package-checks.ts:30-33`), missing-plugin warning (`session-hooks.ts:113-118`), and `/rpiv-setup` installer (`setup-command.ts:43-79`). No edits needed to those three consumer files.
- **Asset shipping**: `package.json` `files` allowlist must include `"prompts/"` or `readFileSync(new URL("./prompts/...", import.meta.url))` breaks in the published tarball. Hard-won via `b9428e9` advisor refactor.
- **`pi-tui` primitives**: `Container`, `Text` (multi-line word-wrapping per width), `Spacer`, `wrapTextWithAnsi`, `truncateToWidth`, `visibleWidth` all exported. `matchesKey(data, Key.up | Key.down | Key.escape)` from `keys.d.ts:165` handles both ANSI and Kitty-protocol key sequences.
- **Idle-gated `appendEntry`** (competitor reference observation): writing to JSONL transcript while main agent streams may interleave entries at confusing positions. Defer with a queue (`Map<sessionFile, BtwAuditRecord[]>`), flush on `agent_end` and `session_before_switch`.

## Decisions

### Decision 1: Direct `completeSimple` over pi-subagents wrapper
Locked in research. `@tintinweb/pi-subagents` has no "share parent messages by reference" mode; closest primitive (`inherit_context: true`) text-serializes branch and spawns a full agent loop. `/btw`'s single-shot semantics map cleanly to `completeSimple(model, {systemPrompt, messages, tools: []}, {apiKey, headers, signal})`. Pattern modeled after `advisor.ts:272-276`.

### Decision 2: Raw `convertToLlm(branch)` messages, NOT serialized prose
Locked in research (line 67-69). Competitor uses raw `Message[]` for prompt-cache parity. Splice shape: `[...convertToLlm(branch), ...sessionHistory.flatMap(h => [user, assistant]), {role:"user", content: question}]`. **Q1 answer locked**: trust provider normalization for adjacent same-role messages at the splice boundary (Anthropic merges silently; document risk for non-Anthropic providers).

### Decision 3: Per-session-file `btwHistory` storage on `globalThis`
**User-driven decision** — observed competitor product persists `/btw` history across `/new` calls, but does NOT persist primary conversation. Most cache-friendly model: per-session-file storage with stable prefix per session.

```ts
const STATE_KEY = Symbol.for("rpiv-btw");
type State = {
  histories: Map<string, BtwTurn[]>;       // key: sessionFile
  snapshots: Map<string, { messages: Message[] }>;  // key: sessionFile
};
const state: State = ((globalThis as any)[STATE_KEY] ??= {
  histories: new Map(),
  snapshots: new Map(),
});
```

Survives module re-import (jiti `moduleCache: false` doesn't purge globalThis). Per-session-keyed → maximum cache hit within one session, no cross-session prefix pollution.

**`BtwTurn` shape — real messages, not fabricated** (iteration 2026-04-18): store the actual `UserMessage` built at call time and the real `AssistantMessage` returned by `completeSimple`. No synthesis, no hardcoded `api`/`provider`/`model`, real `Usage`. Replaying these real objects on the next `/btw` call preserves object-reference stability — the prompt prefix is byte-identical across calls (strengthens Decision 2's cache-parity goal).

```ts
export interface BtwTurn {
  userMessage: UserMessage;       // real, built at call time with timestamp
  assistantMessage: AssistantMessage;  // real, from completeSimple response
}
```

### Decision 4: No audit, no disk persistence — process-scoped only
**User-driven simplification**: history must survive `/new`, `/fork`, `/reload`, `/resume` within the same Pi process, but should NOT survive Pi process restart. The `globalThis`-keyed storage from Decision 3 already satisfies this. Drop `pi.appendEntry("btw:sidechain", ...)` audit, the `pendingAudit` queue, the idle-gating logic, the `agent_end`/`session_before_switch` flush hooks, and the `session_start` JSONL replay.

Trade-off accepted: no audit trail in the session transcript, no cross-process recovery. Matches user's preference for simplicity over durability.

### Decision 5: Cache-safe snapshot on `message_end`
`pi.on("message_end", handler)` filtered: `if (msg.role !== "assistant") return; if (msg.stopReason === "toolUse") return;`. Snapshot the branch via `convertToLlm(getBranch().filter(type==="message").map(e=>e.message))` and store in `state.snapshots.set(sessionFile, {messages})`. Mirrors competitor's `handleStopHooks()` semantics. `/btw` handler prefers snapshot over fresh `getBranch()` for cache-prefix stability; falls back to live `getBranch()` if no snapshot yet.

### Decision 6: Snapshot invalidation on `session_compact` and `session_tree`
**Self-critique-driven**: research line 369 opted out of session-lifecycle handlers, but compaction and branch navigation change branch shape WITHOUT triggering module re-import. Without invalidation, `/btw` would inject pre-compaction content. Add:
```ts
pi.on("session_compact", (_e, ctx) => state.snapshots.delete(getSessionFile(ctx)));
pi.on("session_tree", (_e, ctx) => state.snapshots.delete(getSessionFile(ctx)));
```
`btwHistory` is preserved across these events — it's the user's own asks, not the agent's branch.

### Decision 7: Bottom-slot overlay via `ctx.ui.custom`
`ctx.ui.custom<void>(factory, {overlay: true, overlayOptions: {anchor: "bottom-center", width: "100%", maxHeight: "50%", margin: {left:0, right:0, bottom:0}}})`. Borderless — `Container` + `Text` rows only, no `DynamicBorder`. Three rendering regions: history (dimmed prefix) + current Q (accent prefix) + answer body (default fg) + footer (dim, key hint).

### Decision 8: Own `AbortController` (NOT `ctx.signal`)
`ctx.signal` is bound to main agent (`runner.js:1758`). Forwarding it would couple `/btw` cancellation to main turn. Own controller; abort from overlay's `handleInput` on Esc.

### Decision 9: Overlay `handleInput` for ALL keys
**Self-critique fix** (research line 145 recommended `ctx.ui.onTerminalInput`, but the overlay already captures focus via default `nonCapturing: false`). Use `handleInput(data)` and `matchesKey(data, Key.escape | Key.up | Key.down)` from `@mariozechner/pi-tui`. Single mechanism, no listener-cleanup needed.

### Decision 10: Static system prompt (no `${SIDE_QUESTION}` placeholder)
**Self-critique fix** (research line 217 proposed substitution, but the question is already the final user message). System prompt at `prompts/btw-system.txt` is a static text file describing the side-question role, response style, and "do not call tools" constraint. Loaded once via `readFileSync(new URL("./prompts/btw-system.txt", import.meta.url))`.

**Cross-session pattern hint** appended to system prompt at runtime: a compact summary of the last N (e.g., 10) question-strings from `state.histories` across all sessions. Format: `Recent /btw questions (oldest first): "...", "...", ...`. Lets the model spot patterns when user asks "what have I been asking lately?" without bloating per-call message prefix.

### Decision 11: Refuse `/btw` in non-interactive mode
**Q4 answer locked**. `if (!ctx.hasUI) { ctx.ui.notify("/btw requires interactive mode", "error"); return; }`. Mirrors advisor's `MSG_REQUIRES_INTERACTIVE` at `advisor.ts:63`. The overlay primitive is core to `/btw`'s UX; no fallback.

### Decision 12: No-args `/btw` shows usage
**Q2 answer locked** (matches competitor reference at line 477). `if (!args.trim()) { ctx.ui.notify("Usage: /btw <question>", "warning"); return; }`. Simple, non-destructive.

### Decision 13: Sibling-registration footprint in rpiv-pi
**Locked in research** (lines 333-337). Three edit sites in rpiv-pi:
- `extensions/rpiv-core/siblings.ts` — append one `SIBLINGS` entry
- `package.json` — append one peerDependencies row
- `README.md` — Commands table row at line 147 + sibling bullet at line 182

Auto-pickup: `package-checks.ts`, `setup-command.ts`, `session-hooks.ts`, `pi-installer.ts` consume SIBLINGS data-driven; no edits.

### Decision 14: Minimal hook surface (3 hooks total)
**Locked in research** (lines 369-370) + **simplified per Decision 4**. No tool to strip, no model selection to restore, no audit to flush. Hooks registered: `pi.registerCommand("btw", ...)` + `pi.on("message_end", snapshotHandler)` + `pi.on("session_compact" | "session_tree", invalidateSnapshot)`. That's it.

## Architecture

### rpiv-btw/btw.ts — NEW
Storage state, system prompt loading, message-thread building, `executeBtw`, command registrar, lifecycle registrars.

```typescript
/**
 * @juicesharp/rpiv-btw — /btw side-question slash command.
 *
 * Asks the same primary model a one-off side question using the cloned primary
 * conversation as context. Answer is rendered ephemerally in a bottom-slot
 * overlay (never enters main agent's messages). History persists per-session-file
 * via globalThis-keyed storage; audit via pi.appendEntry("btw:sidechain", ...).
 */

import { readFileSync } from "node:fs";
import { fileURLToPath } from "node:url";
import {
	completeSimple,
	type AssistantMessage,
	type Message,
	type StopReason,
	type UserMessage,
} from "@mariozechner/pi-ai";
import {
	convertToLlm,
	type ExtensionAPI,
	type ExtensionCommandContext,
	type ExtensionContext,
	type SessionEntry,
} from "@mariozechner/pi-coding-agent";
import type { OverlayOptions } from "@mariozechner/pi-tui";
import { showBtwOverlay, type BtwOverlayController } from "./btw-ui.js";

// ---------------------------------------------------------------------------
// Constants — flat named consts, grouped by concern (advisor pattern, b9428e9)
// ---------------------------------------------------------------------------

// Identity
export const BTW_COMMAND_NAME = "btw";

// Storage — globalThis-keyed survives module re-import on /new, /fork, /resume.
// Lost on Pi process exit (intentional — no disk persistence).
export const BTW_STATE_KEY = Symbol.for("rpiv-btw");

// Cross-session pattern hint: how many recent question-strings to inject
export const CROSS_SESSION_HINT_LIMIT = 10;

// Overlay placement — borderless bottom slot, edge-to-edge
export const BTW_OVERLAY_OPTIONS: OverlayOptions = {
	anchor: "bottom-center",
	width: "100%",
	maxHeight: "50%",
	margin: { left: 0, right: 0, bottom: 0 },
};

// Messages (static)
const MSG_REQUIRES_INTERACTIVE = "/btw requires interactive mode";
const MSG_USAGE = "Usage: /btw <question>";
const MSG_NO_MODEL = "/btw requires an active model";

// Errors (static)
const ERR_EMPTY_RESPONSE = "/btw returned no text content.";

// Errors (parameterized)
const errMisconfigured = (label: string, err: string) => `/btw model (${label}) is misconfigured: ${err}`;
const errNoApiKey = (label: string) => `/btw model (${label}) has no API key available.`;
const errCallFailed = (err: string | undefined) => `/btw call failed: ${err ?? "unknown error"}`;
const errCallThrew = (msg: string) => `/btw call threw: ${msg}`;

// ---------------------------------------------------------------------------
// Types
// ---------------------------------------------------------------------------

// Real messages — no fabrication. userMessage is built at call time; assistantMessage
// is the unmodified completeSimple response. Stable object references across calls →
// byte-identical prompt prefix on subsequent /btw invocations (cache parity).
export interface BtwTurn {
	userMessage: UserMessage;
	assistantMessage: AssistantMessage;
}

export interface BtwState {
	histories: Map<string, BtwTurn[]>;
	snapshots: Map<string, { messages: Message[] }>;
}

// ---------------------------------------------------------------------------
// System prompt — loaded once at module init from prompts/btw-system.txt
// ---------------------------------------------------------------------------

export const BTW_SYSTEM_PROMPT = readFileSync(
	fileURLToPath(new URL("./prompts/btw-system.txt", import.meta.url)),
	"utf-8",
).trimEnd();

// ---------------------------------------------------------------------------
// Storage — globalThis-keyed, survives module re-import on /new, /fork, /resume
// (jiti moduleCache:false re-evaluates the module body but does not purge globalThis)
// Mirrors @tintinweb/pi-subagents/src/index.ts:413-422 pattern
// ---------------------------------------------------------------------------

function getState(): BtwState {
	const g = globalThis as unknown as { [k: symbol]: BtwState | undefined };
	let state = g[BTW_STATE_KEY];
	if (!state) {
		state = {
			histories: new Map(),
			snapshots: new Map(),
		};
		g[BTW_STATE_KEY] = state;
	}
	return state;
}

function getSessionFile(ctx: ExtensionContext): string {
	return ctx.sessionManager.getSessionFile() ?? `memory:${ctx.sessionManager.getSessionId()}`;
}

function getSessionHistory(ctx: ExtensionContext): BtwTurn[] {
	const key = getSessionFile(ctx);
	const state = getState();
	let turns = state.histories.get(key);
	if (!turns) {
		turns = [];
		state.histories.set(key, turns);
	}
	return turns;
}

function pushSessionTurn(ctx: ExtensionContext, turn: BtwTurn): void {
	getSessionHistory(ctx).push(turn);
}

export function clearSessionHistory(ctx: ExtensionContext): void {
	getState().histories.set(getSessionFile(ctx), []);
}

function getSnapshot(ctx: ExtensionContext): { messages: Message[] } | undefined {
	return getState().snapshots.get(getSessionFile(ctx));
}

function setSnapshot(ctx: ExtensionContext, snapshot: { messages: Message[] }): void {
	getState().snapshots.set(getSessionFile(ctx), snapshot);
}

export function invalidateSnapshot(ctx: ExtensionContext): void {
	getState().snapshots.delete(getSessionFile(ctx));
}

// Extract text from a UserMessage's content (string | TextContent[] | (TextContent|ImageContent)[]).
export function userMessageText(msg: UserMessage): string {
	if (typeof msg.content === "string") return msg.content;
	return msg.content
		.filter((c): c is { type: "text"; text: string } => c.type === "text")
		.map((c) => c.text)
		.join("\n");
}

// Extract text from an AssistantMessage's content (TextContent | ThinkingContent | ToolCall)[].
// /btw disallows tools and ignores thinking for display; only text matters.
export function assistantMessageText(msg: AssistantMessage): string {
	return msg.content
		.filter((c): c is { type: "text"; text: string } => c.type === "text")
		.map((c) => c.text)
		.join("\n");
}

// Cross-session pattern hint — last N question-strings across ALL sessions.
// Question-strings only (no answers) keeps overhead bounded regardless of history depth.
function getCrossSessionHint(): string {
	const allTurns: { q: string; ts: number }[] = [];
	for (const turns of getState().histories.values()) {
		for (const t of turns) {
			allTurns.push({ q: userMessageText(t.userMessage), ts: t.userMessage.timestamp });
		}
	}
	if (allTurns.length === 0) return "";
	const recent = allTurns
		.sort((a, b) => a.ts - b.ts)
		.slice(-CROSS_SESSION_HINT_LIMIT);
	const lines = recent.map((t, i) => `${i + 1}. ${t.q.replace(/\s+/g, " ").slice(0, 200)}`);
	return `\n\n## Recent /btw questions across sessions (oldest first)\n\n${lines.join("\n")}`;
}

// ---------------------------------------------------------------------------
// Executor — auth, message threading, completeSimple, four StopReason branches
// Modeled after rpiv-advisor/advisor.ts:225-336
// ---------------------------------------------------------------------------

export interface BtwExecResult {
	ok: boolean;
	answer?: string;
	userMessage?: UserMessage;
	assistantMessage?: AssistantMessage;
	error?: string;
	stopReason?: StopReason;
	aborted?: boolean;
}

function readBranchMessages(ctx: ExtensionContext): Message[] {
	const cached = getSnapshot(ctx);
	if (cached) return cached.messages;
	// Cold start (no message_end fired yet) — fall back to live read
	const branch = ctx.sessionManager.getBranch() as SessionEntry[];
	const agentMessages = branch
		.filter((e): e is SessionEntry & { type: "message" } => e.type === "message")
		.map((e) => e.message);
	return convertToLlm(agentMessages);
}

function buildBtwMessages(ctx: ExtensionContext, userMessage: UserMessage): Message[] {
	const branchMessages = readBranchMessages(ctx);
	const history = getSessionHistory(ctx);
	// Real UserMessage + AssistantMessage object references from prior calls — no fabrication.
	// Reusing the same references across calls preserves byte-identical prompt prefix (cache parity).
	const historyMessages: Message[] = history.flatMap((h) => [h.userMessage, h.assistantMessage]);
	// Splice shape per research line 86 + competitor parity:
	// main-thread context first, then /btw conversation chain, then new question.
	return [...branchMessages, ...historyMessages, userMessage];
}

function buildSystemPrompt(): string {
	return BTW_SYSTEM_PROMPT + getCrossSessionHint();
}

export async function executeBtw(
	question: string,
	ctx: ExtensionContext,
	controller: AbortController,
): Promise<BtwExecResult> {
	const model = ctx.model;
	if (!model) {
		return { ok: false, error: MSG_NO_MODEL };
	}
	const modelLabel = `${model.provider}:${model.id}`;

	const auth = await ctx.modelRegistry.getApiKeyAndHeaders(model);
	if (!auth.ok) {
		return { ok: false, error: errMisconfigured(modelLabel, auth.error) };
	}
	if (!auth.apiKey) {
		return { ok: false, error: errNoApiKey(modelLabel) };
	}

	const userMessage: UserMessage = {
		role: "user",
		content: [{ type: "text", text: question }],
		timestamp: Date.now(),
	};
	const messages = buildBtwMessages(ctx, userMessage);
	const systemPrompt = buildSystemPrompt();

	try {
		const response = await completeSimple(
			model,
			{ systemPrompt, messages, tools: [] },
			{
				apiKey: auth.apiKey,
				headers: auth.headers,
				signal: controller.signal, // own AbortController, NOT ctx.signal
			},
		);
		// `response` IS an AssistantMessage (pi-ai/dist/stream.d.ts:7) — store it as-is,
		// no fabrication of api/provider/model/usage.

		if (response.stopReason === "aborted") {
			return { ok: false, aborted: true, stopReason: response.stopReason };
		}
		if (response.stopReason === "error") {
			return {
				ok: false,
				error: errCallFailed(response.errorMessage),
				stopReason: response.stopReason,
			};
		}

		const answerText = assistantMessageText(response).trim();
		if (!answerText) {
			return { ok: false, error: ERR_EMPTY_RESPONSE, stopReason: response.stopReason };
		}

		return {
			ok: true,
			answer: answerText,
			userMessage,
			assistantMessage: response,
			stopReason: response.stopReason,
		};
	} catch (err) {
		// AbortError surfaces here too — translate to aborted result, not error
		const message = err instanceof Error ? err.message : String(err);
		if (controller.signal.aborted) {
			return { ok: false, aborted: true };
		}
		return { ok: false, error: errCallThrew(message) };
	}
}

// ---------------------------------------------------------------------------
// Registrars — pure wiring; handler bodies extracted as named functions
// 3 hooks total: registerCommand + message_end snapshot + compact/tree invalidate
// ---------------------------------------------------------------------------

export function registerMessageEndSnapshot(pi: ExtensionAPI): void {
	pi.on("message_end", async (event, ctx) => {
		const msg = event.message;
		if (msg.role !== "assistant") return;
		if ((msg as AssistantMessage).stopReason === "toolUse") return;
		const branch = ctx.sessionManager.getBranch() as SessionEntry[];
		const agentMessages = branch
			.filter((e): e is SessionEntry & { type: "message" } => e.type === "message")
			.map((e) => e.message);
		setSnapshot(ctx, { messages: convertToLlm(agentMessages) });
	});
}

export function registerInvalidationHooks(pi: ExtensionAPI): void {
	pi.on("session_compact", async (_e, ctx) => invalidateSnapshot(ctx));
	pi.on("session_tree", async (_e, ctx) => invalidateSnapshot(ctx));
}

export function registerBtwCommand(pi: ExtensionAPI): void {
	pi.registerCommand(BTW_COMMAND_NAME, {
		description: "Ask a side question without polluting the main conversation",
		handler: (args: string, ctx: ExtensionCommandContext) => handleBtwCommand(pi, args, ctx),
	});
}

async function handleBtwCommand(
	_pi: ExtensionAPI,
	args: string,
	ctx: ExtensionCommandContext,
): Promise<void> {
	if (!ctx.hasUI) {
		ctx.ui.notify(MSG_REQUIRES_INTERACTIVE, "error");
		return;
	}
	const question = args.trim();
	if (!question) {
		ctx.ui.notify(MSG_USAGE, "warning");
		return;
	}
	if (!ctx.model) {
		ctx.ui.notify(MSG_NO_MODEL, "error");
		return;
	}

	const controller = new AbortController();
	const historySnapshot = [...getSessionHistory(ctx)];

	const { overlayPromise, controllerReady } = showBtwOverlay({
		ctx,
		question,
		history: historySnapshot,
		controller,
		onClearHistory: () => clearSessionHistory(ctx),
	});

	const overlayCtl = await controllerReady;
	const result = await executeBtw(question, ctx, controller);

	if (result.ok && result.answer && result.userMessage && result.assistantMessage) {
		overlayCtl.setAnswer(result.answer);
		pushSessionTurn(ctx, {
			userMessage: result.userMessage,
			assistantMessage: result.assistantMessage,
		});
		// No disk persistence — process-scoped only (Decision 4)
	} else if (result.aborted) {
		// User Esc'd — overlay already dismissed via done(); no further action
	} else if (result.error) {
		overlayCtl.setError(result.error);
	}

	await overlayPromise;
}

### rpiv-btw/btw-ui.ts — NEW
`BtwOverlayController` — implements pi-tui `Component`. Owns rendering state (lines, scrollOffset, status flags). Exposes `setAnswer(text)`, `setError(msg)` for the executor to update mid-await.

```typescript
/**
 * btw-ui — borderless bottom-slot overlay for the /btw side question.
 *
 * State machine: pending → (answer | error). Renders three zones:
 *   1. History (dim) — prior /btw Q+A pairs in this session
 *   2. Current Q (accent) + body (default fg, or red on error)
 *   3. Footer (dim) — key hints
 *
 * Keys via matchesKey from @mariozechner/pi-tui (handles ANSI + Kitty variants):
 *   - Esc → abort in-flight call + dismiss overlay
 *   - ↑/↓ → scroll viewport
 *   - x   → clear current-session /btw history (calls back to btw.ts state)
 */

import type {
	ExtensionCommandContext,
	Theme,
} from "@mariozechner/pi-coding-agent";
import {
	Key,
	matchesKey,
	truncateToWidth,
	wrapTextWithAnsi,
	type Component,
	type TUI,
} from "@mariozechner/pi-tui";
import {
	BTW_OVERLAY_OPTIONS,
	assistantMessageText,
	userMessageText,
	type BtwTurn,
} from "./btw.js";

const FOOTER_HINT = "↑/↓ scroll · x clear history · Esc dismiss";
const PENDING_GLYPH = "…";

type Mode = "pending" | "answer" | "error";

export interface ShowBtwOverlayParams {
	ctx: ExtensionCommandContext;
	question: string;
	history: BtwTurn[];
	controller: AbortController;
	onClearHistory: () => void;
}

export interface ShowBtwOverlayResult {
	overlayPromise: Promise<void>;
	controllerReady: Promise<BtwOverlayController>;
}

export class BtwOverlayController implements Component {
	private mode: Mode = "pending";
	private answer = "";
	private error = "";
	private scrollOffset = 0;
	private history: BtwTurn[];
	private lastWidth = 80;

	constructor(
		private readonly question: string,
		history: BtwTurn[],
		private readonly theme: Theme,
		private readonly tui: TUI,
		private readonly done: (result: void) => void,
		private readonly controller: AbortController,
		private readonly onClearHistory: () => void,
	) {
		this.history = [...history];
	}

	setAnswer(text: string): void {
		this.mode = "answer";
		this.answer = text;
		this.tui.requestRender();
	}

	setError(message: string): void {
		this.mode = "error";
		this.error = message;
		this.tui.requestRender();
	}

	handleInput(data: string): void {
		if (matchesKey(data, Key.escape)) {
			this.controller.abort();
			this.done();
			return;
		}
		if (matchesKey(data, Key.up)) {
			this.scrollOffset = Math.max(0, this.scrollOffset - 1);
			this.tui.requestRender();
			return;
		}
		if (matchesKey(data, Key.down)) {
			const max = Math.max(0, this.allLines(this.lastWidth).length - this.viewportHeight());
			this.scrollOffset = Math.min(max, this.scrollOffset + 1);
			this.tui.requestRender();
			return;
		}
		if (data === "x") {
			this.history = [];
			this.onClearHistory();
			this.scrollOffset = 0;
			this.tui.requestRender();
			return;
		}
	}

	render(width: number): string[] {
		this.lastWidth = width;
		const all = this.allLines(width);
		const vp = this.viewportHeight();
		const slice = all.slice(this.scrollOffset, this.scrollOffset + vp);
		const footer = this.theme.fg("dim", FOOTER_HINT);
		return [
			...slice.map((l) => truncateToWidth(l, width, "...", true)),
			truncateToWidth(footer, width, "...", true),
		];
	}

	invalidate(): void {
		// no-op — render() recomputes from state each cycle
	}

	private allLines(width: number): string[] {
		const lines: string[] = [];

		// Zone 1: history (dim) — prior /btw Q+A pairs in this session
		for (const h of this.history) {
			const q = userMessageText(h.userMessage);
			const a = assistantMessageText(h.assistantMessage);
			lines.push(...wrapTextWithAnsi(this.theme.fg("dim", `Q: ${q}`), width));
			lines.push(...wrapTextWithAnsi(this.theme.fg("dim", `A: ${a}`), width));
			lines.push("");
		}

		// Zone 2a: current question (accent prefix)
		lines.push(...wrapTextWithAnsi(this.theme.fg("accent", `/btw ${this.question}`), width));
		lines.push("");

		// Zone 2b: body — pending / answer / error
		if (this.mode === "pending") {
			lines.push(this.theme.fg("warning", PENDING_GLYPH));
		} else if (this.mode === "error") {
			for (const ln of this.error.split("\n")) {
				lines.push(...wrapTextWithAnsi(this.theme.fg("error", ln.length === 0 ? " " : ln), width));
			}
		} else {
			for (const ln of this.answer.split("\n")) {
				lines.push(...wrapTextWithAnsi(ln.length === 0 ? " " : ln, width));
			}
		}

		return lines;
	}

	private viewportHeight(): number {
		// maxHeight: "50%" of terminal — leave 1 row for footer
		const rows = (this.tui.terminal as { rows?: number }).rows ?? 24;
		return Math.max(4, Math.floor(rows / 2) - 1);
	}
}

export function showBtwOverlay(params: ShowBtwOverlayParams): ShowBtwOverlayResult {
	let resolveReady!: (controller: BtwOverlayController) => void;
	const controllerReady = new Promise<BtwOverlayController>((resolve) => {
		resolveReady = resolve;
	});

	const overlayPromise = params.ctx.ui.custom<void>(
		(tui, theme, _kb, done) => {
			const controller = new BtwOverlayController(
				params.question,
				params.history,
				theme,
				tui,
				done,
				params.controller,
				params.onClearHistory,
			);
			resolveReady(controller);
			return controller;
		},
		{ overlay: true, overlayOptions: BTW_OVERLAY_OPTIONS },
	);

	return { overlayPromise, controllerReady };
}

# @juicesharp/rpiv-btw

Pi extension that adds `/btw <question>` — a side-question slash command that asks the same primary model a one-off question using the cloned primary conversation as context. The answer appears in a borderless bottom-slot overlay; the main agent keeps working.

## Install

```bash
pi install npm:@juicesharp/rpiv-btw
```

Or run `/rpiv-setup` from a Pi session with `@juicesharp/rpiv-pi` loaded — it will install all missing siblings.

## Usage

```
/btw what's the difference between this approach and what we tried in commit a3b2c1d?
```

The overlay opens at the bottom of the terminal with the question echoed and a "…" placeholder while the model thinks. When the answer arrives, it replaces the placeholder.

## Keys

- `↑/↓` — scroll the overlay content
- `x` — clear this session's /btw history
- `Esc` — abort the in-flight call (if still streaming) and dismiss the overlay

## Semantics

- `/btw` runs synchronously even while the main agent is mid-stream. The main agent's turn continues independently.
- The side answer is NEVER added to the main agent's message history — it lives only in the overlay and in process-scoped memory (no transcript audit record).
- History of your `/btw` Q+A pairs threads across calls within the same session, and persists across `/new`, `/fork`, `/reload`, `/resume` within the same Pi process. Lost on Pi process exit (no disk persistence).
- Cross-session pattern analysis: when you ask a question like "what have I been asking about lately?", the last 10 question-strings from all sessions are included as a hint in the system prompt.

## Requires

- A configured primary model (the one you'd chat with normally). `/btw` uses `ctx.model`; no separate model picker.
- Interactive mode. In print/RPC mode, `/btw` will refuse with a notify error.

## License

MIT
